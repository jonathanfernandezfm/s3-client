  `activity-tab.tsx`, `overview-activity-card.tsx`).

From the `/improve deep` session at `6dbaee9` (this round):

- **"Committed production secrets in `.env`"** — false positive.
  `.env` is in `.gitignore:32`; verified `git ls-files | grep .env`
  returns only `.env.example`. The audit subagent saw the file on disk
  and assumed committed.
- **"Unprotected `connections/import` route"** — false positive. The
  route at `src/app/api/connections/import/route.ts:23-130` is
  `withAuth`-wrapped, validates the body shape, checks workspace access
  AND role, enforces tier limits, AND validates each credential with a
  real `ListBuckets` call before saving. Subagent didn't read the file.
- **"Fire-and-forget fetches in crawl/health are bugs"** — downgraded.
  Each call has a `.catch()` handler and the fire-and-forget pattern is
  the documented design (comment at `src/app/api/internal/crawl/route.ts:23`
  says so). Linting style preference, not a bug.
- **"S3 + DB transactional safety on move/copy/rename"** — premise
  wrong. S3 operations cannot participate in a Prisma transaction; the
  right model is compensating actions, not atomicity. Not worth a plan.
- **"AWS SDK is 100 patches behind, plan a minor bump"** — subsumed by
  plan 004 (the same Clerk + AWS SDK bump pulls in the patched
  `fast-xml-parser`). Don't write a separate plan.
- **"Health-probe caching gap"** — health is already cached via React
  Query at the hook layer; the server runs probes on demand. By design.
- **"Crypto / share-link cookie hex validation lacks regex check"**
  (SEC-09 / SEC-03) — real but cosmetic. `Buffer.from(garbage, "hex")`
  returns a short buffer, and length is checked; the worst case is a
  misconfigured env crashes at startup. Not worth a plan unless a follow-
  up appears.
- **"In-memory share-link rate-limit is broken under multi-replica
  deploys"** — real, but requires a product call (do we deploy
  multi-replica? if so, Redis or DB-backed limit). Defer until the
  deployment topology calls for it.
